# TGI / TLM Mobile Development Roadmap

This document tracks the milestones for the Topological General Intelligence (TGI) Android / Mobile implementation.

## Phase 6: Mobile Genesis (IN PROGRESS)
- [ ] **Hardware Awareness**: Map CPU, RAM, and Battery to ^k$ coordinates via `HardwareMapper`.
- [ ] **Action-Coordinate Mapping**: Translate topological paths into system actions (e.g., "notify", "log", "compute").
- [ ] **Android Build (Skeleton)**: Buildozer / Chaquopy configuration for packaging the TGI engine.
- [ ] **English Dictionary Ingestion**: Seed the `KnowledgeMapper` with a full dictionary for linguistic grounding.
- [ ] **Mobile-First Agent**: Create `MobileTGIAgent` for handling Android intent-based queries.

## Phase 7: Full Autonomy & Prototype (PLANNED)
- [ ] **Natural Language Interaction**: Voice-to-TGI lifting using the TLM.
- [ ] **Agentic Action Engine**: Full autonomous task completion based on topological path lifting.
- [ ] **Hardware-Responsive Manifolds**: Dynamic manifold sizing based on device capabilities (e.g., =255$ for flagship, =127$ for mid-range).
- [ ] **Prototype APK Deployment**: Stable, installable APK with full TGI integration.
